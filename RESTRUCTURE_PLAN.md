# TTNB 预判型重构方案

## 核心理念
从"事后总结"转向"事前预判"。每个板块都要回答一个问题：**明天该怎么操作？**

---

## 一、事件日历 — 事件驱动预判

### 现状问题
- 只有CPI/FOMC/LPR等宏观日期，谁都能查到
- 没有产业事件（涨价、投产、政策、财报）
- 没有影响分析

### 重构方案

**数据源升级**：
1. 宏观日历（保留）：CPI/PPI/PMI/FOMC/LPR/MLF + 自动补充影响分析
2. 产业事件（新增）：
   - 东财/同花顺公告抓取：业绩预告/快报、重大合同、停产/复产、涨价/降价、股权变动
   - 产品价格追踪：六氟化钨、稀土、锂、铜等关键品种价格异动
   - 政策日历：国务院/发改委/工信部政策发布计划
3. 财报日历（新增）：未来30天有财报发布的公司，标注市场预期vs实际

**每个事件必须包含**：
```json
{
  "d": "7月10日",
  "e": "6月CPI/PPI数据发布",
  "icon": "📊",
  "s": "宏观",
  "big": 1,
  "impact": {
    "direction": "neutral",        // bullish/bearish/neutral
    "sectors": ["银行","地产","消费"],  // 影响板块
    "logic": "CPI若低于2.5%则降息预期升温，利好银行息差和地产估值",
    "history": "近3次CPI发布后，次日上涨概率67%",
    "stocks": [                     // 受益标的
      {"c":"601318","n":"中国平安","reason":"降息利好保险债配"},
      {"c":"000002","n":"万科A","reason":"降息降低房贷成本"}
    ],
    "watch": "若CPI>3%则通胀压力增大，需警惕"  // 观察要点
  },
  "countdown": 8   // 距今天还有几天
}
```

**前端展示**：
- 按倒计时排序（最近的在最上面）
- 每个事件展开显示：影响方向（红/绿/灰）、影响板块、受益标的、历史统计
- 点击受益标的可直接跳转个股详情

## 二、快讯 — 异动信号监控

### 现状问题
- 收盘后数据被清空
- 只是抓新闻，没有筛选逻辑
- 没有异动信号

### 重构方案

**数据源升级**：
1. 盘前（7:00-9:15）：
   - 隔夜外盘（美股/港股/A50期货）涨跌
   - 重要公告（业绩预告/重大事项/停复牌）
   - 北向资金盘前信号

2. 盘中（9:15-15:00）：
   - 板块异动（5分钟涨幅>2%的板块）
   - 个股异动（涨速异常、量比异常）
   - 北向资金实时流入方向
   - 龙虎榜实时（机构席位出现）

3. 盘后（15:00-22:00）：
   - 龙虎榜完整数据+机构席位分析
   - 大宗交易（溢价/折价、机构专用席位）
   - 融资融券变化
   - 限售解禁

**信号分级**：
```json
{
  "level": "high",           // high/medium/low
  "type": "block_trade",     // 信号类型
  "time": "15:30",
  "title": "大宗交易：寒武纪溢价5.2%成交",
  "detail": "机构专用席位买入2.3亿，卖方为中信证券总部",
  "sectors": ["AI芯片"],
  "stocks": [{"c":"688256","n":"寒武纪"}],
  "signal": "机构溢价大宗买入，通常1-3日内有表现",
  "url": "..."
}
```

**信号类型**：
| 类型 | 说明 | 预判价值 |
|------|------|---------|
| block_trade_premium | 大宗交易溢价 | 机构看好，1-3日 |
| institutional_buy | 龙虎榜机构净买 | 机构建仓，3-5日 |
| northbound_surge | 北向单板块大幅流入 | 外资方向，1日 |
| sector_breakout | 板块5分钟涨>2% | 资金突击，当日 |
| announcement_positive | 业绩预告/合同中标 | 次日大概率涨 |
| price_change | 关键产品涨价/降价 | 产业链影响 |
| policy_release | 政策发布 | 中长期影响 |

**前端展示**：
- 按信号级别排序（high最上面）
- 每条带信号类型标签和预判说明
- 保留7天历史（不再收盘清空）

## 三、简报 — 明日预判

### 现状问题
- AI编的"今天保险涨了7%"——没逻辑
- picks只说"涨了所以推荐"——马后炮
- URL是假的

### 重构方案

**改为"明日预判简报"**，每天17:00生成一次，包含：

```json
{
  "date": "2026-07-03",
  "marketJudgment": {
    "direction": "neutral",      // bullish/bearish/neutral
    "confidence": 60,            // 置信度0-100
    "logic": "今日暴跌后恐慌情绪释放，但无明显企稳信号。北向净流出40亿持续。明日大概率低开震荡，关注2900支撑。",
    "keyLevels": "上证支撑3400/3350，压力3480"
  },
  "watchlist": [
    {
      "c": "688256", "n": "寒武纪",
      "direction": "long",
      "logic": "今日-7.6%超跌，机构大宗溢价买入2.3亿，明日若企稳335可轻仓",
      "entry": "335-338",
      "stop": "328",
      "target": "350",
      "risk": "高"
    }
  ],
  "signals": [
    {
      "type": "northbound_outflow",
      "detail": "北向连续3日净流出，累计-120亿",
      "impact": "短期偏空，关注是否企稳"
    },
    {
      "type": "margin_call",
      "detail": "两融余额下降80亿，杠杆资金离场",
      "impact": "情绪偏弱"
    }
  ],
  "tomorrowEvents": [
    "7月3日 6月财新PMI（预期50.2，前值50.1）"
  ]
}
```

**数据来源**（不再靠AI编）：
1. 当日行情统计（已有）
2. 龙虎榜机构席位分析（已有，需增强解读）
3. 北向资金趋势（已有）
4. 融资融券变化（已有）
5. 大宗交易数据（新增）
6. 技术面关键位（新增，自动计算）
7. 次日事件日历（联动）

**AI只做最后一件事**：把结构化数据组织成自然语言，不做推理判断。

## 实施计划

### 第一阶段：事件日历（预计工作量最大）
1. 重写 `fetch_events.py`：增加产业事件抓取+影响分析
2. 前端事件卡片：展示影响方向/受益标的/历史统计
3. 数据源：东财公告API + 产品价格 + 政策日历

### 第二阶段：快讯异动监控
1. 新建 `signal_monitor.py`：异动信号检测引擎
2. 改造 `news_fetch_once.py`：7×24持续抓取+保留
3. 前端快讯：按信号级别展示+历史保留

### 第三阶段：简报预判
1. 重写 `sentinel_ai.py`：从"总结"改为"预判"
2. 新增数据源：大宗交易+技术面计算
3. 前端简报：预判方向+watchlist+关键位
